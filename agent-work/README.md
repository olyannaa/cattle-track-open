# Agent Work

Эта папка фиксирует, как в проекте Cattle Track использовались AI-агенты и Codex при технической разработке. Она нужна не как презентационный артефакт, а как инженерный след: какие задачи делегировались агентам, как проверялись результаты и какие правила должны соблюдать следующие агенты.

## Что лежит в папке

- [CODEX_WORKFLOW.md](CODEX_WORKFLOW.md) - процесс работы с Codex: постановка задачи, изменения, проверки, коммиты.
- [AGENT_HANDOFF.md](AGENT_HANDOFF.md) - краткий handoff для следующего AI-агента или разработчика.
- [TASK_LOG.md](TASK_LOG.md) - журнал задач, выполненных с участием AI-агентов.

Связанные документы:

- [../agents.md](../agents.md) - общая инструкция для AI-агентов в репозитории.
- [../docs/ai-development/AI_USAGE.md](../docs/ai-development/AI_USAGE.md) - описание использования AI-инструментов в разработке.
- [../docs/ai-development/AGENT_TASK_LOG.md](../docs/ai-development/AGENT_TASK_LOG.md) - расширенный журнал AI-задач.

## Принципы

- Агент не является источником истины: изменения проверяются сборкой, тестами, скриптами или визуальной проверкой.
- Важные выводы фиксируются в документации рядом с кодом.
- Результат работы агента должен быть воспроизводимым по git diff, коммитам и проверкам.
- Секреты, токены и реальные приватные credentials не записываются в репозиторий.
